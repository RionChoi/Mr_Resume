## 2026-04-04

### 1. Issue where sub-content is not displayed after mouse hover

    기존에 프로젝트 리스트가 제대로 펼쳐지지 않았던 이유는 크게 세 가지 기술적인 충돌 때문이었습니다.

    1. max-height 애니메이션의 한계
    기존 코드는 max-h-0에서 max-h-[600px]로 변하는 방식을 사용했습니다. CSS에서 max-height는 값이 명확하게 지정되어야 애니메이션이 동작하는데, 만약 내용이 600px을 넘어가거나 브라우저가 높이를 계산하는 시점과 충돌이 나면 애니메이션이 멈추거나 아예 실행되지 않는 경우가 빈번합니다.

    2. IntersectionObserver와의 충돌
    리스트 아이템 각각에 .fade-up 클래스가 걸려 있었는데, 이 클래스는 처만이 투명도(opacity: 0)와 위치 이동(translateY) 속성을 가지고 있었습니다.

    문제: 화면에 나타날 때 적용되는 이 '등장 애니메이션'과 마우스를 올렸을 때의 '확장 애니메이션'이 하나의 transition-all 속성 안에서 서로 엉키면서, 브라우저가 어떤 애니메이션을 우선시해야 할지 판단하지 못해 멈춰버린 것이었습니다.
    3. Pointer Events(마우스 인식) 오류
    아이템이 위로 올라오면서 나타나는(.fade-up) 도중에 유저가 마우스를 올리면, 실제 요소의 위치와 브라우저가 인식하는 마우스의 위치가 일치하지 않아 onMouseEnter 이벤트가 제대로 발생하지 않는 현상이 있었습니다.

    ✅ 해결 방법 (Framer Motion 도입)
    이러한 복합적인 문제를 해결하기 위해 Framer Motion 라이브러리로 로직을 완전히 교체했습니다.

    유연한 높이 계산: height: "auto"를 사용하여 내용의 길이에 상관없이 정확한 높이를 계산해 펼쳐줍니다.
    애니메이션 분리: 등장 애니메이션(whileInView)과 호버 확장 애니메이션을 독립적으로 관리하여 충돌을 원천 차단했습니다.
    안정적인 반응: 마우스 이동 속도가 빨라도 에러 없이 즉각적으로 반응하도록 설계했습니다.
